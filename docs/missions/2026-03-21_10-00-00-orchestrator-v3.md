## Dependencias entre agentes (v3)

```
01-infra  → agrega `cuenta` a interface Operation + CSV headers + data-loader parser
    └→ habilita que 02-logica pueda agrupar por cuenta en getYieldsData

02-logica → actualiza getYieldsData para agrupar por cuenta + columna Balance
          → agrega Cuentas server action
    └→ habilita 03-ui para renderizar dashboard con cuentas
    └→ habilita 03-ui para mostrar modal onEdit real

03-ui     → selector Cuenta + brokers nuevos en TradeForm
          → YieldsGrid recibe columnas dinámicas por cuenta
          → onEdit modal real + onView modal
          → Cuentas UI section
          → Sidebar (si tiempo)

04-qa     → actualiza tests por nuevos campos (cuenta)
          → E2E trades.spec.ts completo
```

**Orden recomendado:**
1. **Agente 01** (Cuenta en datos)
2. **Agente 02** (dashboard por cuenta + Balance + Cuentas CRUD)
3. **Agente 03** (UI: form campos + YieldsGrid + modales)
4. **Agente 04** (tests + E2E + CI)
