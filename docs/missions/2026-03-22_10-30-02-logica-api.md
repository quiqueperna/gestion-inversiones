## TAREA 2 — `getTradeUnits()` (ex-`getTrades()`)

La función ya existente enriquece Trade Units abiertas con precio Yahoo Finance.
Actualizar referencias de campos:

```typescript
export async function getTradeUnits() {
    const state = ensureDataLoaded();

    const openUnits = state.tradeUnits.filter(tu => tu.status === 'OPEN');
    if (openUnits.length === 0) return state.tradeUnits;

    // Fetch precios actuales para TUs abiertas en paralelo
    const priceResults = await Promise.all(
        openUnits.map(tu => getCurrentPriceWithFallback(tu.symbol))
    );
    const today = new Date();

    return state.tradeUnits.map(tu => {
        if (tu.status === 'CLOSED') return tu;
        const idx = openUnits.indexOf(tu);
        const currentPrice = priceResults[idx] ?? tu.entryPrice;
        const exitAmount = currentPrice * tu.qty;
        const days = Math.max(1, Math.ceil((today.getTime() - new Date(tu.entryDate).getTime()) / (1000 * 60 * 60 * 24)));
        const pnlNominal = exitAmount - tu.entryAmount;
        const pnlPercent = tu.entryAmount > 0 ? (pnlNominal / tu.entryAmount) * 100 : 0;
        const tna = (pnlPercent / days) * 365;
        return {
            ...tu,
            exitPrice: currentPrice,
            exitAmount,
            exitDate: today,
            days,
            pnlNominal,
            pnlPercent,
            tna,
        };
    });
}
```

## TAREA 4 — `getOpenExecutionsForClosing(symbol, side, account, broker)`

La función actual `getOpenOperationsForClosing(symbol, closingType)` filtra solo por símbolo.
La nueva versión **debe filtrar también por account y broker** (Regla de Aislamiento):

```typescript
export async function getOpenExecutionsForClosing(
    symbol: string,
    side: 'BUY' | 'SELL',
    account: string,
    broker: string
) {
    const oppositeSide = side === 'SELL' ? 'BUY' : 'SELL';
    const state = ensureDataLoaded();
    return state.executions.filter(exec =>
        exec.symbol === symbol.toUpperCase() &&
        exec.side === oppositeSide &&
        exec.account === account &&      // ← NUEVO: aislamiento por account
        exec.broker === broker &&        // ← NUEVO: aislamiento por broker
        !exec.isClosed &&
        exec.remainingQty > 0
    ).map(exec => ({
        id: exec.id,
        date: exec.date,
        symbol: exec.symbol,
        qty: exec.remainingQty,
        price: exec.price,
        amount: exec.remainingQty * exec.price,
        broker: exec.broker,
        account: exec.account,
        side: exec.side,
        days: Math.ceil((new Date().getTime() - new Date(exec.date).getTime()) / (1000 * 60 * 60 * 24))
    }));
}
```

## TAREA 7 — `createExecution(data)` (ex-`createOperation`)

```typescript
export async function createExecution(data: {
    date: string;
    symbol: string;
    qty: number;
    price: number;
    broker: string;
    account: string;
    side: 'BUY' | 'SELL';
    currency?: string;
    commissions?: number;
    isFalopa?: boolean;
    isIntra?: boolean;
}) {
    const state = ensureDataLoaded();
    const newExec: Execution = {
        id: state.executions.length + 1,
        date: new Date(data.date + 'T12:00:00'),
        symbol: data.symbol.toUpperCase(),
        qty: data.qty,
        price: data.price,
        amount: Math.abs(data.qty * data.price),
        broker: data.broker,
        account: data.account || 'USA',
        side: data.side,
        currency: data.currency || 'USD',
        commissions: data.commissions || 0,
        remainingQty: Math.abs(data.qty),
        isClosed: false,
        isFalopa: data.isFalopa || false,
        isIntra: data.isIntra || false,
    };
    state.executions.push(newExec);
    return newExec;
}
```

## MOTOR DE MATCHING — ESTRATEGIAS (para futuras sesiones pero documentar ahora)

Según `trade-units.md` sección 4.II, las estrategias de asignación son:
1. **FIFO** — cierra la unidad más antigua (default actual, ya implementado)
2. **LIFO** — cierra la unidad más reciente
3. **Max Profit** — cierra la unidad con mayor ganancia porcentual actual
4. **Min Profit** — cierra la unidad con peor rendimiento actual
5. **Manual Match** — el usuario vincula manualmente a una `unit_id` específica (ya implementado vía `closeTradeUnitManually`)

Para esta sesión de refactor: **implementar FIFO + Manual** (los ya existentes, adaptados).
LIFO, MaxProfit, MinProfit quedan como pendiente P2 para próxima sesión.
