model Execution {
  id           Int      @id @default(autoincrement())

  amount       Float

  updatedAt    DateTime @updatedAt

  id           Int      @id @default(autoincrement())
  symbol       String
  qty          Float
  account      String
  broker       String

  entryPrice   Float
  entryDate    DateTime
  entryAmount  Float

  exitExecId   Int?
  exitPrice    Float?
  exitDate     DateTime?
  exitAmount   Float?

  pnlNominal   Float    @default(0)
  pnlPercent   Float    @default(0)
  tna          Float    @default(0)

  entryExecution Execution @relation("EntryExecution", fields: [entryExecId], references: [id])

  createdAt    DateTime @default(now())

  @@index([symbol])
  @@index([status])

## ERRORES A EVITAR (de lessons.md)

1. **No olvidar actualizar `resetMemoryState()`** — al cambiar los keys del memoryState, `resetMemoryState()` debe reflejar la nueva estructura o los tests de integración contaminan estado entre runs.
2. **Campos opcionales en interfaces** — cuando `exitDate`, `exitPrice`, etc. pasan a opcionales, hacer grep de TODOS los usos antes de cambiar la interface para identificar guards necesarios.
3. **`any` implícito** — no usar `any` en las interfaces nuevas. YieldsGrid y componentes que consumen TradeUnits deben tener types explícitos.
4. **Fechas** — `new Date("YYYY-MM-DD")` nunca. Siempre `new Date(str + 'T12:00:00')`.
