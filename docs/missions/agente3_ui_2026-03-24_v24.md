# Agente 3 — UI/UX
**Sesión:** v24 | **Fecha:** 2026-03-24

## Objetivo
Crear la página y formulario de login con soporte email/password y Google OAuth.

## Archivos a crear
1. **`src/app/login/page.tsx`** — Server Component que renderiza el formulario
2. **`src/app/auth/callback/route.ts`** — Route Handler para el callback de OAuth (Google)

## Especificaciones

### login/page.tsx
- Client Component (`'use client'`)
- Formulario con campos: email, password (con placeholder)
- Botón "Iniciar sesión" que llama a Server Action `login`
- Botón "Continuar con Google" que llama a `loginWithGoogle`
- Link "¿No tenés cuenta? Registrate" → formulario de signup alternativo (toggle)
- Mostrar errores de formulario si los hay (estado local con `useFormState` o `useState`)
- Diseño: dark mode glassmorphism, compacto, centrado en pantalla
- Colores: zinc-900 background, botón Google blanco con icono SVG

### auth/callback/route.ts
- GET handler: intercambia `code` por sesión con `supabase.auth.exchangeCodeForSession(code)`
- Redirige a `/` al éxito, a `/login?error=...` si falla

## Reglas UI
- Dark mode glassmorphism
- Inputs con placeholder obligatorio
- Sin emojis
- Español

## Dependencias
- Agente 2 debe haber creado las Server Actions.
