# Misión: Agente UI — v21 (2026-03-23)

## Objetivo
Rediseñar ImportCSVView con flujo multi-paso: preview de trades, flow MANUAL interactivo, confirmación final.

## Steps del flujo

### step = 'upload'
Igual que antes: drag-drop, parse CSV, dos tablas (válidas/inválidas), botón "Calcular trades"

### step = 'preview'
Llama previewBulkImport(validRows)
Muestra:
- **Sección colapsable "Ejecuciones a importar"** (N): lista de filas válidas
- **Sección colapsable "Trades que se generarán"** (N): tabla ProjectedTrade con columnas:
  Symbol | Side | Qty | Entrada | Salida | PnL | % | Días | Cuenta | Estrategia | Estado
- Si hay manualMatchRequired → botón "Resolver coincidencias manuales (N)" que lleva a step='manual'
- Si no hay manual → botón "Confirmar importación"

### step = 'manual'
Para cada ManualMatchRequired (una por una, con paginación 1/N):
- Muestra datos de la SELL: Symbol, Fecha, Qty, Precio
- Tabla de candidatos BUY (misma UI que TradeForm individual pero en modo lista)
- Usuario elige un BUY candidato y cantidad a cerrar
- "Siguiente" / "Omitir"
- Al terminar todos → step = 'final-review'

### step = 'final-review'
Igual que 'preview' pero con las decisiones manuales incluidas.
Re-llama previewBulkImport con manualDecisions.
Botón "Confirmar importación definitiva"

### step = 'done'
Banner de éxito. Botón cerrar.

## Props nuevas
- `previewFn`: server action previewBulkImport
- `confirmFn`: server action confirmBulkImportWithTrades

## Collapsible component
Usar estado local `collapsed: boolean` por sección.
Header con chevron y contador de items.
