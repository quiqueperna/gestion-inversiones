## Dependencias entre agentes (v2)

```
01-infra  (prices.ts real + USE_DB flag)
    └→ habilita precios reales en getOpenPositions (trades.ts)

02-logica (CashFlow memoryState + deleteOp + updateOp + equityCurve + Funcionalidades.md)
    └→ habilita handlers reales en 03-ui
    └→ resetMemoryState() habilita tests de 04-qa

03-ui     (CloseTradeModal + CashFlowForm + filtros + dashboard completo + export)
    └→ depende de 01 y 02 para tener las actions completas

04-qa     (tests integración + E2E + regresión parser)
    └→ puede correr tests unitarios en paralelo con 01 y 02
    └→ E2E depende de 03 (UI completa)
```

**Orden recomendado:**
1. Agente 02 (agrega `resetMemoryState`, CashFlow en memoria, deleteOp, equityCurve, Funcionalidades.md)
2. Agentes 01 y 04-unitarios **en paralelo** (prices.ts real + tests integración)
3. Agente 03 (UI: modal, form, filtros, dashboard)
4. Agente 04-E2E al final
