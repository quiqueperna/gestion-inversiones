## ERRORES A EVITAR (de lessons.md)

1. **Leer los 3 archivos de UI antes de implementar cualquier listado** — UI.md, UI-List.md, UI-Behavior.md. (lessons.md v9 Error 1)
2. **`v || <span>` no es ReactNode válido en TypeScript strict** — usar `v ? String(v) : <span>—</span>`. (lessons.md v9 Error 2)
3. **FilterBar: excluir SOLO en vistas de formulario puro** — `nueva-exec`, `ie`, `cuentas`, `brokers`. Las vistas de listado siempre tienen FilterBar. (lessons.md v9 Error 3)
4. **Fechas de server actions** — siempre verificar `instanceof Date` antes de `.slice(0,10)`. Helper recomendado: `toDateStr(d)`. (lessons.md v8 Error 2)
5. **No usar `setLoading(true)` en re-fetches** — usar `setRefreshing(true)` para no desmontar la UI. (lessons.md v3 Error 2)
6. **Condiciones negativas compuestas** — evitar `view !== "X" && !Y`. Usar variables semánticas nombradas. (lessons.md v7 Error 1)
7. **Verificar que server action devuelve los campos que referencia la UI** — `item.account`, `item.entryDate`, etc. (lessons.md v7 Error 2)
