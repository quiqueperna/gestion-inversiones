## Objetivo
Verificar que la implementación de persistencia cumple con los estándares de calidad del proyecto.

## Checklist

### 1. TypeScript
```bash
npx tsc --noEmit
```
- Debe dar 0 errores
- Corregir cualquier error antes de avanzar

### 2. ESLint
```bash
npm run lint
```
- Verificar que los guardrails de ESLint siguen activos
- `src/app/**` y `src/lib/**` NO deben importar `@prisma/client` ni `@/server/**`
- Solo `src/server/**` puede importar Prisma

### 3. Tests unitarios
```bash
npm run test
```
- Los 42 tests deben pasar
- Si alguno falla por el cambio de async en `ensureDataLoaded()`, corregir el mock correspondiente

### 4. Build de producción
```bash
npm run build
```
- Debe terminar sin errores

## Estándares del proyecto
- Fechas: `new Date(str + 'T12:00:00')` siempre — nunca `new Date("YYYY-MM-DD")`
- Server Actions: solo en `src/server/actions/`
- UI: no accede a Prisma directamente
- TypeScript strict: sin `any` implícito donde sea posible
