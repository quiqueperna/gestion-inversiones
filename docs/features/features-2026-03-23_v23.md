## Reordenamiento de columnas — Pantalla Importar Ejecuciones CSV

### 1. Tabla de ejecuciones a importar

Nuevo orden de columnas: **ID, Fecha Entrada, Símbolo, Lado, Cantidad, Precio, Broker, Cuenta**

- "ID" (antes "#") — número secuencial de fila
- Símbolo ahora precede a Lado
- Headers usan `SortIcon` del hook `useSortable` (activo/inactivo correctamente)

### 2. Tabla de trades a generar

Nuevo orden de columnas: **F. Entrada, Símbolo, Lado, Cantidad, P. Entrada, P. Salida, F. Salida, Días, PnL $, PnL %, TNA, Estado, Broker, Cuenta**

Columnas nuevas respecto a v22:
- **Lado**: muestra "Compra" para todos los trades (posiciones long)
- **F. Salida**: fecha y hora de cierre (`exitDate` de `ProjectedTrade`), `—` si trade abierto
- **TNA**: tasa nominal anual (`tna = (pnlPercent/days)*365`), `—` si trade abierto

Cambios técnicos:
- `TradeRow` extiende `ProjectedTrade` con `_exitDate: Date | null`
- Eliminado doble estado (`useSortable + useState`) — ahora solo `useSortable`
- Headers usan `SortIcon` inline del hook
