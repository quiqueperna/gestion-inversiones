## Resumen
Corrección de bug crítico de persistencia: brokers, cuentas, ejecuciones y trades no se guardaban en la BD al usar la UI. Causas raíz identificadas y corregidas. Brokers y Cuentas ahora tienen tablas propias en Supabase.

---

## Bug Raíz 1 — `isDBBacked=false` en runtime
`ensureLoaded()` en `transactions.ts` usaba `initializeMemoryState()` (CSV) que setea `isDBBacked=false`. Si se invocaba antes que `ensureDataLoaded()` de trades/dashboard, el flag quedaba en `false` y NINGUNA mutación escribía a la BD (el `if (state.isDBBacked)` las bloqueaba todas).

**Fix:** `ensureLoaded()` reemplazado por versión `async` que carga desde DB exactamente igual que trades.ts y dashboard.ts.

## Bug Raíz 2 — Sin modelos Prisma para Broker y Cuenta
Brokers y Cuentas no tenían tabla en la BD. Se perdían al reiniciar el servidor.

**Fix:** Modelos `Broker` y `Cuenta` agregados al schema + migración + seed de defaults.

## Funcionalidades Implementadas

### 1. Modelos `Cuenta` y `Broker` en Prisma
- **Archivo:** `prisma/schema.prisma`
- `Cuenta`: id, nombre (unique), descripcion, matchingStrategy, timestamps
- `Broker`: id, nombre (unique), descripcion, timestamps
- Migración: `20260323205606_add_broker_cuenta_v17`

### 2. Seed de brokers y cuentas por defecto
- 3 cuentas: USA, Argentina, CRYPTO
- 8 brokers: Schwab, Binance, Cocos, Balanz, AMR, IOL, IBKR, PP

### 3. `initializeFromDB()` — parámetros opcionales cuentas/brokers
- **Archivo:** `src/lib/data-loader.ts`
- Acepta arrays de cuentas y brokers desde Prisma
- Usa defaults hardcoded como fallback si no se proveen

### 4. `ensureLoaded()` en `transactions.ts` → async + DB first
- **Archivo:** `src/server/actions/transactions.ts`
- Carga executions, tradeUnits, cashFlows, cuentas y brokers desde DB
- Idéntica lógica que trades.ts y dashboard.ts → garantiza `isDBBacked=true` siempre

### 5. `ensureDataLoaded()` en trades.ts y dashboard.ts — carga cuentas/brokers
- Ambas funciones ahora también cargan `db.cuenta.findMany()` y `db.broker.findMany()`
- Se pasan a `initializeFromDB()` para que el memoryState tenga datos reales de BD

### 6. Persistencia Cuentas (CRUD completo)
- `addMemoryCuenta()` → `db.cuenta.create()` + sync id de BD a memoria
- `removeMemoryCuenta()` → `db.cuenta.delete()`
- `updateMemoryCuenta()` → `db.cuenta.update()`
- `updateCuentaMatchingStrategy()` → `db.cuenta.update({ matchingStrategy })`

### 7. Persistencia Brokers (CRUD completo)
- `addMemoryBroker()` → `db.broker.create()` + sync id de BD a memoria
- `removeMemoryBroker()` → `db.broker.delete()`
- `updateMemoryBroker()` → `db.broker.update()`
