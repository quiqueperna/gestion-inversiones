## Fix: Unique constraint en Account/Broker por userId

### Problema
Al crear una cuenta/broker, Prisma lanzaba `P2002: Unique constraint failed on the fields: ('nombre')`.
El constraint viejo `Account_nombre_key` / `Broker_nombre_key` era un UNIQUE INDEX (no un CONSTRAINT),
por lo que el `ALTER TABLE ... DROP CONSTRAINT` de la migración v25 no lo eliminó.

### Fix aplicado
```sql
DROP INDEX IF EXISTS "Account_nombre_key";
DROP INDEX IF EXISTS "Broker_nombre_key";
CREATE UNIQUE INDEX IF NOT EXISTS "Account_userId_nombre_key" ON "Account"("userId", "nombre");
CREATE UNIQUE INDEX IF NOT EXISTS "Broker_userId_nombre_key" ON "Broker"("userId", "nombre");
```

### Resultado
Dos usuarios distintos ahora pueden tener cuentas/brokers con el mismo nombre sin conflicto.
El aislamiento por userId funciona correctamente.
