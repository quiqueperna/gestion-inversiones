# Features Implementadas — Sesión v16
**Fecha:** 2026-03-23
**Hora:** 17:46
**Versión:** 16

## Resumen
Activación de persistencia real con Supabase/Prisma. El sistema migró de "Modo Demo" (datos en memoria desde CSV) a persistencia real en base de datos PostgreSQL local (Supabase).

---

## Funcionalidades Implementadas

### 1. Schema de Prisma actualizado
- **Archivo:** `prisma/schema.prisma`
- Agregado campo `exchange_rate Float @default(1)` en modelo `Execution`
- Agregado campo `cuenta String?` en modelo `CashFlow`
- Agregado campo `exitExecId Int?` + relación `exitExec` en modelo `TradeUnit`
- Agregada relación inversa `exitTradeUnits` en modelo `Execution`

### 2. Migración de base de datos
- **Migration:** `prisma/migrations/20260323203503_add_missing_fields_v16/`
- Migración aplicada exitosamente al Supabase local (puerto 54322)

### 3. Seed de datos de prueba
- **Archivo:** `prisma/seed-v16.ts`
- Carga completa desde CSV (`public/data/initial_operations.csv`): 338 operaciones
- FIFO matching para generar TradeUnits
- 3 executions mock (posiciones abiertas: TSLA, AAPL, NVDA)
- 9 CashFlows mock (3 brokers × 3 movimientos)
- **Resultados en BD:** 341 Executions, 201 TradeUnits, 9 CashFlows

### 4. Función `initializeFromDB()` en data-loader
- **Archivo:** `src/lib/data-loader.ts`
- Nueva función que carga datos desde Prisma directamente al memoryState
- Mapea correctamente los tipos Prisma → interfaces TypeScript
- Setea `isDBBacked: true` para habilitar persistencia en mutaciones

### 5. Flag `isDBBacked` en memoryState
- **Archivo:** `src/lib/data-loader.ts`
- `isDBBacked: false` — estado cargado desde CSV (tests, fallback)
- `isDBBacked: true` — estado cargado desde Supabase/Prisma
- Las mutaciones solo persisten a DB cuando `isDBBacked` es true (evita conflictos en tests)

### 6. `ensureDataLoaded()` migrado a async con carga desde DB
- **Archivos:** `src/server/actions/trades.ts`, `src/server/actions/dashboard.ts`
- Intenta cargar desde Prisma primero (si hay datos en BD)
- Fallback a CSV si la BD está vacía

### 7. Persistencia en mutaciones de Execution
- **Archivo:** `src/server/actions/trades.ts`
- `createExecution()` → `db.execution.create()`
- `deleteExecution()` → `db.execution.delete()`
- `updateExecution()` → `db.execution.update()`

### 8. Persistencia en mutaciones de TradeUnit
- **Archivo:** `src/server/actions/trades.ts`
- `deleteTradeUnit()` → `db.tradeUnit.delete()`
- `closeTradeUnitManually()` → `db.$transaction([create exec, update exec, upsert tradeUnit])`
- `closeTradeUnitWithQuantity()` → `Promise.all([múltiples create/update en DB])`

### 9. Persistencia en mutaciones de CashFlow
- **Archivo:** `src/server/actions/transactions.ts`
- `addMemoryCashFlow()` → `db.cashFlow.create()` + sync a memoria
- `removeMemoryCashFlow()` → `db.cashFlow.delete()` + sync a memoria
- `updateMemoryCashFlow()` → `db.cashFlow.update()` + sync a memoria
- `createCashFlow()` ya usaba Prisma — actualizado para sincronizar también memoria y soportar campo `cuenta`
- `deleteCashFlow()` — actualizado para sincronizar también memoria
