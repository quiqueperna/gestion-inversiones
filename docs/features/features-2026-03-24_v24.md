# Features Implementadas — Sesión v24
**Fecha:** 2026-03-24 | **Hora:** 13:20 | **Versión:** 24

## Autenticación completa con Supabase SSR

### 1. Server Actions de autenticación (`src/server/actions/auth.ts`)
- `login(formData)`: inicio de sesión con email y contraseña
- `signup(formData)`: registro con confirmación por email
- `logout()`: cierre de sesión + redirect a `/login`
- `loginWithGoogle()`: OAuth con Google vía Supabase

### 2. Protección de rutas en Middleware (`src/middleware.ts`)
- Usuarios no autenticados son redirigidos a `/login`
- Rutas públicas excluidas: `/login`, `/auth/callback`
- Se mantiene el refresco de token en cada request

### 3. Página de login (`src/app/login/page.tsx`)
- Formulario email/contraseña con toggle login/signup
- Botón "Continuar con Google" con ícono SVG oficial
- Feedback de errores y mensajes de éxito inline
- Dark mode glassmorphism (zinc-900/zinc-800)
- Textos en español

### 4. Auth callback handler (`src/app/auth/callback/route.ts`)
- GET route que intercambia `code` por sesión Supabase
- Redirige a `/` al éxito, a `/login?error=...` si falla

### 5. Clientes Supabase (pre-existentes, verificados)
- `src/utils/supabase/server.ts` — cliente SSR para Server Components y Server Actions
- `src/utils/supabase/client.ts` — cliente browser para Client Components

## Configuración requerida (manual en Supabase Dashboard)
1. Habilitar proveedor Google en Authentication > Providers
2. Agregar `http://localhost:3000/auth/callback` en Redirect URLs
3. Variables de entorno: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
