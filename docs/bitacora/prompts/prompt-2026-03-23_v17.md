# Bitácora de Prompts

## 23 de Marzo, 2026 — Sesión v17

### Prompt 1
> "brokers, cuentas, ejecucion y trades no se estan persistiendo cuando se usa la UI"

**Resultado:**
Bug crítico corregido. Dos causas raíz identificadas: (1) `ensureLoaded()` en transactions.ts inicializaba desde CSV fijando `isDBBacked=false`, bloqueando todas las escrituras a BD cuando se llamaba primero; (2) Brokers y Cuentas no tenían modelos en Prisma (solo vivían en memoria). Fix: ensureLoaded convertido a async + DB first, modelos Broker/Cuenta agregados al schema con migración y seed, CRUD completo de brokers/cuentas ahora persiste en Supabase. TypeScript OK, 42/42 tests, build limpio.
